# Google Docs API セットアップガイド

このアプリケーションを使用するには、Google Docs APIの認証情報が必要です。

## 手順

### 1. Google Cloud Projectの作成

1. [Google Cloud Console](https://console.cloud.google.com/)にアクセス
2. 新しいプロジェクトを作成（または既存のプロジェクトを選択）
3. プロジェクト名を入力（例：`kindle2md`）

### 2. Google Docs APIの有効化

1. 左メニューから「APIとサービス」→「ライブラリ」を選択
2. 検索ボックスで「Google Docs API」を検索
3. 「Google Docs API」をクリックし、「有効にする」をクリック

### 3. 認証情報の作成

1. 左メニューから「APIとサービス」→「認証情報」を選択
2. 「認証情報を作成」→「OAuthクライアントID」を選択
3. 同意画面の構成を求められた場合：
   - ユーザータイプ：「外部」を選択
   - アプリ名：任意の名前を入力
   - ユーザーサポートメール：自分のメールアドレス
   - デベロッパーの連絡先情報：自分のメールアドレス
   - 保存して続行
   - スコープは追加不要（スキップ）
   - テストユーザーに自分のメールアドレスを追加
4. OAuthクライアントIDの作成：
   - アプリケーションの種類：「デスクトップアプリ」を選択
   - 名前：任意の名前（例：`Kindle2MD Desktop`）
   - 「作成」をクリック
5. JSONファイルをダウンロード

### 4. 認証情報ファイルの配置

1. ダウンロードしたJSONファイルを `credentials.json` にリネーム
2. このファイルを `credentials/` ディレクトリに配置

```bash
mv ~/Downloads/client_secret_*.json ./credentials/credentials.json
```

### 5. 初回実行時の認証

初めてアプリケーションを実行すると、ブラウザが開いてGoogleアカウントへのログインを求められます。

1. ログインして、アプリケーションへのアクセスを許可
2. 「このアプリは確認されていません」と表示された場合：
   - 「詳細」をクリック
   - 「[アプリ名]に移動（安全ではないページ）」をクリック
3. アクセスを許可すると、`token.json` が自動的に作成されます

これで設定完了です！

## トラブルシューティング

### 「このアプリは確認されていません」エラー

これは自分で作成したアプリなので問題ありません。上記の手順で回避できます。

### 認証情報が見つからないエラー

`credentials/credentials.json` ファイルが正しく配置されているか確認してください。

### アクセス権限エラー

Google Docs APIが有効化されているか確認してください。
